# PlayMCP 위치 기반 카카오톡 알림 설정

> 참고: 이 문서는 Codex/ChatGPT가 켜져 있을 때 동작하는 PlayMCP 방식입니다. PC나 Codex 앱이 꺼져 있어도 매일 오전 8시에 자동 발송하려면 `docs/SERVER_CRON_KAKAO_SETUP.md`의 Vercel Cron + 카카오 REST API 방식을 사용하세요.

이 프로젝트는 카카오 공식 OAuth 토큰을 직접 관리하지 않고, ChatGPT에 연결된 PlayMCP 카카오톡 도구로 메시지를 보낸다.

## 핵심 구조

```text
휴대폰 브라우저에서 위치 저장
→ 홈페이지 서버가 좌표 저장
→ 매일 오전 8시 ChatGPT 자동화 실행
→ 저장된 좌표로 날씨 조회
→ PlayMCP 카카오톡 도구로 나에게 보내기
```

## 왜 위치 저장이 필요한가

카카오톡이나 PlayMCP는 수신자 휴대폰의 GPS 위치를 자동으로 읽을 수 없다.

그래서 사용자가 카카오톡을 받을 휴대폰에서 홈페이지를 열고, 브라우저 위치 권한을 한 번 허용해야 한다.

## 사용자 설정 방법

1. `LOCATION_UPDATE_TOKEN` 환경변수를 길고 예측하기 어려운 값으로 설정한다.
2. 카카오톡을 받을 관리자 휴대폰에서 홈페이지를 연다.
3. `관리자 위치 저장 토큰`에 `LOCATION_UPDATE_TOKEN` 값을 입력한다.
4. 토큰이 저장된 브라우저에서는 페이지 진입 시 자동으로 위치 권한을 요청한다.
5. 위치 권한을 허용한다.
6. 위치 이름을 바꾸고 싶으면 `집`, `회사`, `서울 성동구`처럼 입력한 뒤 `관리자 위치 저장`을 누른다.
7. 저장 후 홈페이지의 지역과 좌표가 바뀌었는지 확인한다.

브라우저 보안 정책상 권한 허용 없이 위치를 몰래 읽을 수는 없다. 자동 수집은 "페이지 진입 시 권한 요청을 자동으로 띄우고, 사용자가 허용하면 저장"하는 의미다.

공개 링크는 누구나 열람할 수 있지만, 위치 변경은 관리자 토큰이 있는 브라우저에서만 가능하다.

## 구현된 API

| 경로 | 역할 |
| --- | --- |
| `/api/location` | 저장된 위치 조회 및 휴대폰 위치 저장 |
| `/api/weather/current` | 저장된 위치 기준 오늘 날씨와 우산 판단 반환 |

`POST /api/location`은 `x-location-update-token` 헤더가 `LOCATION_UPDATE_TOKEN`과 일치해야 저장된다.

## PlayMCP 자동화 메시지

PlayMCP 도구는 메시지 길이가 200자 이하이므로, 자동화 메시지는 짧게 만든다.

예시:

```text
[오늘의 우산 알림]
지역: 집
판단: 우산 필요
15:00 전후 비 가능성. 최고 27°C / 최저 21°C / 강수확률 최대 70%
```

## 운영 주의사항

* 위치 정보는 위도/경도 좌표로 저장된다.
* 휴대폰 위치가 바뀌면 관리자 휴대폰에서 홈페이지를 다시 열거나 `관리자 위치 저장`을 눌러 갱신한다.
* PlayMCP 발송은 홈페이지 서버가 직접 수행하지 않고, ChatGPT 자동화가 수행한다.
* 홈페이지에서 위치와 날씨를 계산하고, 자동화가 그 결과를 읽어 PlayMCP로 전송한다.
* 링크를 public으로 열면 다른 사람도 페이지를 볼 수 있지만, 관리자 토큰 없이는 대표 위치를 바꿀 수 없다.
