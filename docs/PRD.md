# PRD: 매일 아침 우산 필요 여부 판단 및 카카오톡 알림 Agent

## 1. 제품 개요

본 제품은 매일 아침 사용자의 지정 지역 날씨를 자동으로 수집하고, 당일 우산 필요 여부를 판단한 뒤, 홈페이지를 업데이트하고 오전 8시에 카카오톡으로 요약 알림을 발송하는 자동화 Agent이다.

사용자는 별도로 날씨 앱을 확인하지 않아도 "오늘 우산을 챙겨야 하는지"를 빠르게 판단할 수 있으며, 홈페이지에서는 당일 날씨 요약, 시간대별 강수 가능성, 우산 필요 판단 근거를 확인할 수 있다.

---

## 2. 문제 정의

사용자는 아침 출근·등교 전 짧은 시간 안에 날씨를 확인해야 한다. 하지만 일반 날씨 정보는 기온, 강수확률, 강수량, 하늘상태 등 정보가 분산되어 있어 실제 행동 판단, 즉 "우산을 가져갈지 말지"를 빠르게 결정하기 어렵다.

따라서 본 Agent는 날씨 데이터를 단순 표시하는 것을 넘어, 강수확률·강수형태·강수량·시간대 정보를 종합해 우산 필요 여부를 명확하게 판단하고, 사용자가 확인하기 쉬운 채널인 홈페이지와 카카오톡으로 전달한다.

---

## 3. 목표

### 3.1 핵심 목표

* 매일 오전 8시 이전 최신 날씨 데이터를 자동 수집한다.
* 당일 우산 필요 여부를 "필요 / 권장 / 불필요"로 판단한다.
* 판단 결과와 근거를 홈페이지에 자동 업데이트한다.
* 매일 오전 8시에 카카오톡으로 요약 메시지를 발송한다.
* 데이터 수집 실패 또는 발송 실패 시 예외 처리 및 재시도 로직을 제공한다.

### 3.2 비목표

* 전국 모든 사용자를 대상으로 한 대규모 서비스 운영
* 장기 예보 기반 여행 일정 추천
* 실시간 재난 알림 서비스
* 기상청 수준의 자체 예보 모델 개발
* 위치 기반 자동 추적 기능

## 4. 타깃 사용자

### 4.1 1차 사용자

* 매일 출근·등교 전 날씨 확인이 필요한 개인 사용자
* 특정 지역의 당일 우산 필요 여부를 빠르게 알고 싶은 사용자

### 4.2 2차 사용자

* 회사, 연구실, 동아리, 소규모 커뮤니티 등에서 아침 공지 형태로 날씨 알림을 받고 싶은 사용자
* 건물 로비, 사무실 홈페이지, 내부 포털에 당일 날씨 정보를 표시하고 싶은 운영자

## 5. 사용자 시나리오

### 시나리오 1: 개인 사용자

1. 사용자는 거주 지역 또는 출근 지역을 사전에 등록한다.
2. Agent는 매일 아침 기상청 API에서 해당 지역의 당일 예보를 수집한다.
3. Agent는 강수확률, 강수형태, 예상 강수량을 기준으로 우산 필요 여부를 판단한다.
4. 홈페이지가 자동으로 업데이트된다.
5. 오전 8시에 카카오톡으로 "오늘 우산 필요 여부" 메시지가 발송된다.
6. 사용자는 카카오톡 메시지만 보고 우산을 챙길지 결정한다.

### 시나리오 2: 커뮤니티 운영자

1. 운영자는 특정 지역을 기준으로 날씨 Agent를 설정한다.
2. 매일 홈페이지에 오늘의 날씨 카드가 자동 게시된다.
3. 오전 8시에 구성원에게 카카오톡 알림이 발송된다.
4. 구성원은 홈페이지 링크를 통해 시간대별 상세 정보를 확인한다.

## 6. 핵심 기능 요구사항

### 6.1 날씨 데이터 수집

#### 설명

기상청 API 또는 신뢰 가능한 날씨 API에서 지정 지역의 당일 예보 데이터를 수집한다.

#### 입력 데이터

* 지역명
* 위도/경도 또는 기상청 격자 좌표
* 조회 날짜
* 기준 시간

#### 수집 항목

* 시간대별 강수확률
* 강수형태
* 예상 강수량
* 하늘상태
* 기온
* 습도
* 풍속
* 발표 시각

#### 요구사항

* 매일 오전 7시 40분~7시 55분 사이 데이터를 수집한다.
* API 응답 실패 시 최소 2회 재시도한다.
* 최신 데이터가 없을 경우 직전 발표 데이터를 사용하되, 메시지에 "직전 예보 기준"임을 표시한다.
* 데이터 수집 결과는 DB에 저장한다.

---

### 6.2 우산 필요 여부 판단

#### 설명

수집된 날씨 데이터를 기반으로 당일 우산 필요 여부를 자동 판단한다.

#### 판단 등급

* 필요: 우산을 반드시 챙기는 것이 좋음
* 권장: 비 가능성이 있어 휴대용 우산 권장
* 불필요: 우산 없이 이동 가능

#### 1차 판단 로직

| 조건 | 판단 |
| --- | --- |
| 당일 시간대 중 강수형태가 비/눈/소나기인 경우 | 필요 |
| 강수확률 60% 이상인 시간대가 1개 이상 있는 경우 | 필요 |
| 강수확률 40~59%인 시간대가 2개 이상 있는 경우 | 권장 |
| 예상 강수량이 1mm 이상인 경우 | 필요 |
| 오전/퇴근 시간대 강수확률 40% 이상 | 권장 이상 |
| 강수확률 30% 미만이며 강수형태 없음 | 불필요 |

#### 판단 보정 로직

* 출근 시간대 07:00~10:00, 퇴근 시간대 17:00~20:00의 강수 가능성은 가중치를 높인다.
* 강수확률은 낮지만 강수형태가 "비"로 표시되면 최소 "권장"으로 판단한다.
* 기상특보 또는 호우 관련 정보가 있는 경우 "필요"로 판단한다.
* 불확실성이 높은 경우 "권장"으로 판단하여 보수적으로 안내한다.

#### 출력 예시

* "오늘은 오후 3시 이후 비 예보가 있어 우산이 필요합니다."
* "강수확률은 높지 않지만 퇴근 시간대 소나기 가능성이 있어 휴대용 우산을 권장합니다."
* "오늘은 강수 예보가 없어 우산이 필요하지 않습니다."

---

### 6.3 홈페이지 자동 업데이트

#### 설명

우산 필요 여부와 날씨 요약 정보를 홈페이지에 자동 반영한다.

#### 표시 정보

* 오늘 날짜
* 기준 지역
* 우산 필요 여부
* 한 줄 요약
* 시간대별 강수확률
* 최고/최저 기온
* 현재 또는 예보 기준 하늘상태
* 데이터 기준 시각
* 카카오톡 발송 여부

#### 화면 구성

##### 상단 카드

* 제목: "오늘 우산 필요할까?"
* 상태 뱃지: 필요 / 권장 / 불필요
* 요약 문장: "오후부터 비 예보가 있어 우산이 필요합니다."

##### 상세 영역

* 시간대별 날씨 표
* 강수확률 그래프
* 판단 근거
* 마지막 업데이트 시간

#### 요구사항

* 매일 오전 8시 이전 자동 업데이트되어야 한다.
* 업데이트 실패 시 이전 데이터를 그대로 보여주되, "업데이트 실패" 상태를 표시한다.
* 모바일에서도 확인 가능한 반응형 UI로 구현한다.

---

### 6.4 카카오톡 알림 발송

#### 설명

매일 오전 8시에 사용자의 카카오톡으로 우산 필요 여부와 날씨 요약을 발송한다.

#### 메시지 구성

```text
[오늘의 우산 알림]

지역: 서울 성동구
판단: 우산 필요 ☔

오후 3시 이후 비 예보가 있습니다.
최고기온 27°C / 최저기온 21°C
강수확률 최대 70%

상세 보기: 홈페이지 링크
```

#### 요구사항

* 매일 오전 8시에 자동 발송한다.
* 메시지 발송 전 홈페이지 업데이트가 완료되어야 한다.
* 발송 실패 시 1분 간격으로 최대 3회 재시도한다.
* 실패 로그를 저장한다.
* 카카오 API 토큰 만료 시 재인증 또는 관리자 알림을 발생시킨다.

#### MVP 발송 방식

* 1차 MVP: 카카오톡 "나에게 보내기" 방식
* 확장 버전: 친구에게 보내기 또는 알림톡/비즈메시지 연동 검토

---

### 6.5 관리자 설정

#### 설명

운영자가 지역, 발송 시간, 판단 기준 등을 설정할 수 있도록 한다.

#### 설정 항목

* 지역명
* 위도/경도 또는 기상청 격자 좌표
* 알림 발송 시간
* 카카오톡 수신 대상
* 우산 판단 기준
* 홈페이지 제목
* 메시지 템플릿
* API 키 관리

#### 요구사항

* 환경변수 또는 관리자 페이지에서 설정 가능해야 한다.
* API 키와 토큰은 코드에 직접 저장하지 않는다.
* 민감 정보는 암호화하거나 보안 저장소에 저장한다.

## 7. 시스템 아키텍처

### 7.1 구성 요소

#### Scheduler

* 매일 정해진 시간에 작업 실행
* 예: cron, GitHub Actions, Cloud Scheduler, AWS EventBridge

#### Weather Collector

* 기상청 API 호출
* 지역 좌표 기반 예보 데이터 수집
* 응답 데이터 정규화

#### Decision Engine

* 우산 필요 여부 판단
* 판단 근거 생성
* 요약 문장 생성

#### Homepage Updater

* 정적 페이지 또는 DB 기반 홈페이지 업데이트
* 최신 날씨 카드 반영

#### Kakao Sender

* 카카오톡 메시지 API 호출
* 발송 성공/실패 처리
* 토큰 관리

#### Database / Storage

* 날씨 원본 데이터
* 판단 결과
* 발송 로그
* 에러 로그 저장

---

#### MVP

* Backend: Python FastAPI 또는 Node.js Express
* Scheduler: cron 또는 GitHub Actions

## 9. 배치 실행 흐름

### 매일 아침 실행 순서

1. 오전 7시 45분: Scheduler 실행
2. Weather Collector가 날씨 API 호출
3. 응답 데이터 검증 및 저장
4. Decision Engine이 우산 필요 여부 판단
5. 홈페이지 데이터 업데이트
6. 오전 8시: 카카오톡 메시지 발송
7. 발송 결과 로그 저장
8. 실패 시 재시도 또는 관리자 알림

## 10. 예외 처리

### 10.1 날씨 API 실패

#### 상황

* API 서버 오류
* 네트워크 오류
* 인증키 오류
* 최신 예보 미제공

#### 처리

* 최대 2회 재시도
* 실패 시 직전 성공 데이터를 사용
* 홈페이지에 "최신 예보 수집 실패" 표시
* 카카오톡 메시지에 "직전 예보 기준" 표시

---

### 10.2 카카오톡 발송 실패

#### 상황

* 토큰 만료
* 권한 부족
* API 제한 초과
* 수신자 정보 오류

#### 처리

* 최대 3회 재시도
* 토큰 만료 시 갱신 시도
* 갱신 실패 시 관리자 이메일 또는 로그 알림
* 홈페이지에는 업데이트 상태 유지

---

### 10.3 판단 불가

#### 상황

* 강수확률 데이터 누락
* 강수형태 데이터 누락
* 일부 시간대 예보 누락

#### 처리

* 남아 있는 데이터를 기준으로 판단
* 불확실성이 높으면 "권장"으로 표시
* 판단 근거에 "일부 데이터 누락" 표시

* 날씨 수집 및 발송 로그 저장
* 실패 시 재시도

### 12.2 MVP에서 제외

* 여러 사용자별 지역 설정
* 위치 기반 자동 지역 감지
* 관리자 대시보드
* 장기 예보 분석
* AI 자연어 대화형 질의
* 카카오톡 단체방 자동 발송

## 14. 보안 및 개인정보

* 카카오 API 토큰은 환경변수 또는 Secret Manager에 저장한다.
* 사용자 카카오 식별 정보는 최소한으로 저장한다.
* 위치 정보는 동/구 단위까지만 저장하는 것을 기본으로 한다.
* 로그에는 액세스 토큰, 리프레시 토큰, API 키를 저장하지 않는다.
* 관리자 페이지가 생길 경우 인증 기능을 필수 적용한다.

---

## 15. 주요 정책 및 제약사항

* 카카오톡 메시지 API는 권한 및 수신 대상 제약이 있으므로, 초기 MVP에서는 본인 계정 발송을 우선 구현한다.
* 여러 사용자에게 발송하려면 카카오 친구 API 권한, 서비스 가입 구조, 알림톡/비즈메시지 사용 여부를 별도로 검토해야 한다.
* 날씨 API 데이터는 발표 시각에 따라 최신성이 달라질 수 있으므로, 메시지에 기준 시각을 반드시 포함한다.
* 오전 8시 정시 발송을 위해 날씨 수집은 최소 5~15분 전에 실행한다.

## 20. MVP 완료 기준

다음 조건을 만족하면 MVP 완료로 본다.

* 매일 오전 8시에 카카오톡 메시지가 자동 발송된다.
* 홈페이지에 당일 우산 필요 여부가 자동 반영된다.
* 우산 판단 결과가 "필요 / 권장 / 불필요" 중 하나로 표시된다.
* 메시지에 지역, 판단 결과, 판단 근거, 홈페이지 링크가 포함된다.
* 날씨 API 또는 카카오톡 발송 실패 시 로그가 남는다.
* 최소 7일 이상 연속 정상 동작을 검증한다.
